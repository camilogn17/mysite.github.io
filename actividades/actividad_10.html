En HTML5 no es necesario proporcionar valores para todos los atributos, con el elemento <video> . Estos atributos son como interruptores de encedido/apagado. Si el atributos esta presente, activa esa opción. Si se omite el atributo, la opción está desactivada. Si el navegador no admite el elemento <video> o el formato de video </video> de cierre.

preload: este atributo le indica al navegador qué hacer cuando se carga la página. Puede tener uno de tres valores:
none: el navegador no debe cargar el video hasta que el usuario presiona Play.
auto: el navegador debe descargar el video cuando se carga la página.
metadata: ek navegador debe simplemente recopilar información como el tamaño, el primer fotograma, la lista de pistas y la duración.

La etiqueta video: Para especificar la ubicación del archivo que se reproducirá, puede usar el elemento <source> dentro del elemento <video>. (Esto debe reemplazar el atributo src en la etiqueta <video> de apertura.) También puede utilizar varios elementos <source> para especificar que el video está disponible en diferentes formatos. (Debido a un error en el ipad, debe proporcionar el video MP4 como el primer formaro. De lo contrario, puede que no inicie.)

src: este atributo especifica la ruta de acceso al video.

type: se debe utilizar este atributo para decirle al navegador que formato es el video. De lo contrario, descargará parte dek video para ver si puede reproducir el archivo (que tomára tiempo y ancho de banda).

codecs: el codec que se usó para codificar el video se suministra dentro del atributo type. Tenga en cuenta el uso de comillas simples, asi como comillas dobles en el atributo del tipo, cuando se suminstra

<audio> HTML5 introdujo el elemento <audio> para incluir archivos de audio en sus paginas. Al igual que con el video HTML5, los navegadores esperan diferentes formaros para el audio. El elemento <audio> tiene un número de atributos que le permiten controlar la reproducción de audio:

src: este atributo especifica la ruta de acceso al archivo de audio.

controls: este atributo indica si el repoductor debe mostrar los controles. Si no utiliza este atributo, no se mostrarán controles de forma predeterminada. También puede especificar sus propios controles mediante JavaScript.

autoplay:la presencia de este atributo indica que el audio debe empezar a reproducirse automaticamente. (Se considera una mejor prática permitir que los visitantes elijan reproducir audio.)

loop: este atributo especifica que la pista de audio debe reproducirse de nuevo una vez que haya finalizado

HTML YouTube Videos

YouTube video ID YouTube mostrará un ID (como EF81Pb5IWoU), al guardar (o reproducir) un video. Puede utilizar este ID y hacer referencia a su video en el código HTML.

Reproducción de un vídeo de YouTube en HTML.

para reproducir el video en una página web, haga lo siguiente:
  • Sube el video a YouTube
  • Anote el ID de video
  • Defina un elemento <iframe> en su página web
  • Deje que el atributo src apunte a la URL del video
  • Utilice los atributos width y height para especificar la dimensión del reproductor
